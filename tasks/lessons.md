## Lessons Learned

- Always read and follow project-local `AGENTS.md` before continuing implementation.
- For non-trivial tasks, update `tasks/todo.md` first with a checkable plan and keep it in sync during execution.
- Do not claim completion until focused tests and full-suite verification are run and reported.
- If any roadmap requirement is only partially implemented (for example SIGHUP behavior), treat it as unfinished work and close it before declaring the phase complete.
